# NotesDesk Client — NAS API 瘦客户端

连接群晖 / 本机 `notes-desk-server` 的桌面浮窗，**不运行本地钉钉桥**，数据与提醒均在 NAS 端处理。

## 目录

| 子目录 | 平台 | 说明 |
|--------|------|------|
| [`mac/`](mac/) | macOS 13+ | Swift + SwiftUI 浮窗 |
| [`win/`](win/) | Windows 10+ | WPF + 系统托盘 |

## 前置条件

1. NAS 已运行 `notes-desk-server`（见 [安装与配置指南](../notes-desk-server/docs/安装与配置指南.md)）
2. 浏览器可访问 `http://<NAS_IP>:17887/health` 返回 `{"ok":true,...}`
3. 客户端 **设置** 填写 Web 用户名/密码（默认 `admin`/`admin`），或填写与 Docker 一致的 `NOTESDESK_TOKEN`

> 仅填服务器地址会出现 `HTTP 401 Unauthorized`，因 `/api/*` 需登录或 Token。

## macOS

```bash
cd notes-desk-client/mac
./run.sh
```

首次在 **设置** 中填写：

- **服务器地址**：如 `http://192.168.1.100:17887`（不要末尾斜杠）
- **Web 用户名/密码**：默认 `admin` / `admin`（登录后建议在 Web 改密码）
- **API Token**（可选）：若 Docker 设置了 `NOTESDESK_TOKEN`，可只填 Token 不填密码

## Windows

```bash
cd notes-desk-client/win
dotnet run --project NotesDeskClient.Win
```

或使用 `run.bat`。

## 架构

```
手机钉钉 → notes-desk-server (Docker, 7×24)
                ├── SQLite
                ├── 钉钉 Stream 桥
                └── 到期提醒

Mac/Win notes-desk-client
    └── HTTP 轮询 + 操作 API
        GET  /health
        GET  /api/tasks
        PATCH /api/tasks/:id
        DELETE /api/tasks/completed
```
