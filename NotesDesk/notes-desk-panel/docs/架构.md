# NotesDesk 架构与开发文档

> 项目总览见 [NotesDesk README](../../README.md)。

## 1. 技术栈

| 层级 | 选型 |
|------|------|
| 语言 | Swift 5.9+ |
| UI | SwiftUI + AppKit（NSPanel 浮窗） |
| 入口 | MenuBarExtra（菜单栏 `note.text`） |
| 读数据 | AppleScript → Notes.app |
| 同步触发 | FSEvents 监听 Notes 数据目录 |
| 配置 | UserDefaults |
| 构建 | Swift Package Manager（`swift build`） |
| 沙盒 | **无**（SPM 本地构建，FSEvents 可监听 Notes 目录） |

## 2. 模块划分

```
Sources/NotesDesk/
├── NotesDeskApp.swift              # @main，MenuBarExtra，应用生命周期
├── App/
│   └── FloatingPanelController.swift  # NSPanel 浮窗管理
├── Models/
│   └── NotesFolder.swift           # 文件夹模型
├── Services/
│   ├── NotesReader.swift           # AppleScript 读文件夹/第一条笔记
│   ├── NotesWatcher.swift          # FSEvents 监听
│   └── NotesOpener.swift           # AppleScript 打开指定笔记
├── ViewModels/
│   └── PanelViewModel.swift        # 状态、刷新、配置
├── Views/
│   ├── FloatingPanelView.swift     # 浮窗 UI
│   ├── WindowDragLayer.swift       # AppKit 透明拖动层
│   └── SettingsView.swift          # 设置（透明度、字号）
└── Utilities/
    └── AppSettings.swift           # UserDefaults 封装
```

## 3. 数据流

```
┌─────────────┐     AppleScript      ┌──────────────┐
│  Notes.app  │ ◄─────────────────── │ NotesReader  │
└─────────────┘                      └──────┬───────┘
       │ 写入 DB                             │ NotePreview
       ▼                                    ▼
┌─────────────┐     FSEvents         ┌──────────────┐
│ group.com.  │ ───────────────────► │ PanelViewModel│
│ apple.notes │   防抖 0.5s          └──────┬───────┘
└─────────────┘                             │ @Published
                                            ▼
                                   ┌──────────────┐
                                   │ FloatingPanel│
                                   │     View     │
                                   └──────────────┘
```

**单进程架构**：菜单栏、浮窗 UI、AppleScript、FSEvents 均在同一 `NotesDesk` 进程内。不存在单独的 Host 应用。

## 4. 关键实现

### 4.1 NotesReader

- `listFolders()`：遍历 `accounts` → `folders`，返回 `NotesFolder` 列表（含 account、name、noteCount）。
- `fetchFirstNote(in:)`：定位文件夹，取 `item 1 of notes`，返回 `title`、`body`（plaintext）、`id`。
- 使用 `\u{001F}` 拼接 AppleScript 返回值，避免正文含分隔符时解析错误。

### 4.2 NotesWatcher

- 监听路径：`~/Library/Group Containers/group.com.apple.notes`
- `FSEventStream` + `kFSEventStreamCreateFlagFileEvents`
- 防抖 **0.5s**，回调主线程触发 `PanelViewModel.refresh()`
- 无沙盒限制，可可靠接收文件系统事件

### 4.3 FloatingPanelController

- `NSPanel`：`nonactivatingPanel`、`canJoinAllSpaces`、`hidesOnDeactivate = false`
- **窗口层级**：`normalWindow - 1`（低于普通 App 窗口，切换 App 时被遮挡；高于 `desktopIconWindow`，以便响应拖动）
- `isOpaque = false`，SwiftUI 侧半透明背景
- 关闭窗口仅隐藏，不退出 App

### 4.4 WindowDragLayer（PanelDragLayerView）

- AppKit 透明 overlay 盖在浮窗最上层
- 标题栏区域：`trackEvents` 手动 `setFrameOrigin` 拖动（桌面层级下 `performDrag` 不可靠）
- 正文 ScrollView 区域：事件穿透，仍可滚动
- 拖动结束防抖保存窗口位置到 UserDefaults

### 4.5 PanelViewModel

- `@MainActor`，`ObservableObject`
- 属性：`note`、`isLoading`、`errorMessage`、`selectedFolder`、`panelOpacity`、`fontSize`、`panelVisible`
- `bootstrap()`：`NotesWatcher.start(onChange: refresh)` + 初始 `loadFolders()` / `refresh()`
- **不做定时轮询**，仅 FSEvents + 菜单栏「立即刷新」

### 4.6 打开笔记

- `NotesOpener.open(noteID:)`：`tell application "Notes" to show note id "..."`
- 浮窗内**双击**正文触发

## 5. 配置项（UserDefaults）

| Key | 类型 | 默认 |
|-----|------|------|
| `selectedFolder` | String? | nil |
| `panelOpacity` | Double | 0.92 |
| `fontSize` | Double | 14 |
| `panelOriginX` / `panelOriginY` | Double | 100 |
| `panelVisible` | Bool | true |

## 6. 构建与运行

```bash
cd notes-desk-panel
./run.sh
```

或：

```bash
swift build -c release
.build/release/NotesDesk
```

首次运行：**系统设置 → 隐私与安全性 → 自动化**，允许 NotesDesk 控制「备忘录」。

## 7. 风险与限制

| 风险 | 缓解 |
|------|------|
| AppleScript 较慢 | 仅 FSEvents 触发时刷新，不轮询 |
| 文件夹重名 | 取 account + name 匹配；MVP 取第一个匹配 |
| Notes 数据库路径变更 | 系统大版本升级后验证 FSEvents 路径 |
| FSEvents 漏事件 | 菜单栏「立即刷新」 |

## 8. 与小组件版对比（开发选型）

| | 浮窗版（本项目） | 小组件版 |
|--|------------------|----------|
| 自动更新机制 | FSEvents → 同进程 UI | Host 轮询 5s + Widget 时间线 10s |
| 为何更快 | 无沙盒 + 无跨进程 | Widget 扩展沙盒 + reload 限流 |
| 菜单栏 note.text | 本 App 控制面板 | Host 的数据同步入口（非展示层） |

详见 [开发文档 §5](../../docs/开发文档.md#5-常见问题与原因说明)。
