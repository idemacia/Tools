# NotesDesk 需求说明

> 项目总览见 [NotesDesk README](../../README.md)。

## 1. 背景

macOS 自带「备忘录」的**文件夹桌面小组件**无法完整显示第一条备忘录内容（仅预览、固定尺寸、不可滚动）。NotesDesk 为自用工具，在桌面以**浮窗**形式只读展示指定文件夹中**按备忘录 App 默认排序的第一条**备忘录全文。

浮窗版是仓库中的**首选日常方案**：单进程、无沙盒，FSEvents 可近实时刷新，正文可滚动。

## 2. 目标用户

- 本人自用，不上架 App Store

## 3. 功能需求

### 3.1 核心功能（MVP）

| ID | 需求 | 优先级 |
|----|------|--------|
| F-01 | 用户可选择备忘录中的**文件夹** | P0 |
| F-02 | 浮窗展示该文件夹**默认排序第 1 条**备忘录的标题 + 正文（plaintext） | P0 |
| F-03 | 正文支持几百字完整显示，超出时可滚动 | P0 |
| F-04 | 用户在系统「备忘录」中修改内容后，浮窗**自动同步**（FSEvents，非定时轮询） | P0 |
| F-05 | 双击浮窗可在系统「备忘录」中打开对应条目 | P1 |
| F-06 | 菜单栏（note.text）控制：显示/隐藏浮窗、选择文件夹、退出 | P0 |

### 3.2 明确不做

- 不在浮窗内编辑备忘录
- 不支持锁定/加密备忘录
- 不做 WidgetKit 小组件（见 [`notes-desk-widget`](../../notes-desk-widget/) 独立子项目）
- 不做固定间隔轮询（依赖 FSEvents + 菜单栏手动刷新）
- 不做 iOS / 多设备同步逻辑（依赖系统备忘录自身 iCloud 同步）

## 4. 非功能需求

| ID | 需求 |
|----|------|
| NF-01 | macOS 13+，本机运行 |
| NF-02 | 首次运行需授予「自动化 → 备忘录」权限 |
| NF-03 | 配置（文件夹、窗口位置、透明度等）本地持久化 |
| NF-04 | 浮窗置于 `normalWindow - 1` 层级：切换 App 时被遮挡、不抢焦点，且可拖动 |

## 5. 「第一条」定义

与备忘录 App 列表一致：读取 AppleScript `item 1 of notes of folder`，顺序由 App 内该文件夹的**默认排序**决定（通常为「编辑日期，最新优先」）。

## 6. 验收标准

1. 选择文件夹后，浮窗显示该文件夹排序第一的标题与正文。
2. 在备忘录 App 修改该条内容并保存后，浮窗在**约 1 秒内**自动更新，无需手动刷新。
3. 几百字正文完整可见；更长内容可通过滚动查看。
4. 双击浮窗可跳转到系统备忘录对应笔记。
5. 菜单栏可隐藏浮窗、切换文件夹、退出应用。

## 7. 权限与隐私

- **自动化**：通过 AppleScript 读取备忘录、打开指定笔记。
- **文件访问**：监听 `~/Library/Group Containers/group.com.apple.notes` 目录变化以触发刷新（无 App 沙盒，FSEvents 有效）。
- 数据不离开本机，无网络请求。

## 8. 与小组件版的关系

| 对比项 | 浮窗版（本项目） | 小组件版 |
|--------|------------------|----------|
| 进程 | 单 App | Host + Widget Extension |
| 菜单栏 note.text | 同一 App 的控制入口 | Host 的数据同步入口 |
| 自动更新 | FSEvents，近实时 | Host 轮询约 5s + Widget 时间线约 10s |
| 正文 | 可滚动全文 | 固定尺寸截断 |

浮窗版**不需要**单独的 Host 进程——菜单栏、浮窗、读备忘录均在 **NotesDesk 一个 App** 内完成。详见 [开发文档 §4、§5.3](../../docs/开发文档.md#4-架构对比)。
