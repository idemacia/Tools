# NotesDesk Server — 安装与配置指南（用户向）

> 面向**群晖 NAS** 使用者：导入 Docker 镜像 → 创建容器 → Web 配置钉钉与大模型。  
> 开发者 build 镜像见 [群晖Docker打包与部署.md](./群晖Docker打包与部署.md)。

---

## 1. 你需要准备什么

| 项 | 说明 |
|----|------|
| 群晖 NAS | DSM 7+，已安装 **Container Manager** |
| 网络 | NAS 能访问外网（钉钉 Stream 需出站 HTTPS） |
| 钉钉 | 企业账号，能在 [钉钉开放平台](https://open.dingtalk.com/) 创建**企业内部应用** |
| 大模型（可选） | OpenAI 或**兼容 OpenAI API** 的服务（如 DeepSeek、通义等） |

---

## 2. 安装方式（二选一）

### 方式 A：导入预构建镜像 tar（推荐，无需 build）

适合从 [GitHub Releases](https://github.com/idemacia/Tools/releases) 下载 `notesdesk-server.tar` 的用户。

1. 下载 `notesdesk-server.tar`（amd64，适用于多数 x86 群晖）
2. 上传到 NAS，例如 `/volume1/docker/notesdesk/notesdesk-server.tar`
3. SSH 或 Container Manager 导入：

```bash
sudo docker load -i /volume1/docker/notesdesk/notesdesk-server.tar
sudo docker images | grep notesdesk
```

4. 创建数据目录：

```bash
sudo mkdir -p /volume1/docker/notesdesk/data/config
sudo mkdir -p /volume1/docker/notesdesk/data/store
sudo chown -R 999:999 /volume1/docker/notesdesk/data
```

5. 创建并启动容器：

```bash
sudo docker run -d \
  --name notesdesk \
  --restart unless-stopped \
  -p 17887:8080 \
  -e TZ=Asia/Shanghai \
  -e DATA_DIR=/data \
  -v /volume1/docker/notesdesk/data:/data \
  notesdesk-server:latest
```

6. 浏览器访问 `http://<NAS_IP>:17887`，进入 **第 3 节** 配置。

详细 tar 流程见 [release/README.md](../release/README.md)。

### 方式 B：从源码自行 build

见 [群晖Docker打包与部署.md](./群晖Docker打包与部署.md)（Mac 上 buildx → save tar → 群晖 load）。

---

## 3. 首次登录 Web

1. 打开 `http://<NAS_IP>:17887`
2. 默认账号：**admin** / **admin**
3. 登录后 → **设置** → **账号与安全** → **立即修改密码**

---

## 4. 配置钉钉机器人

完整开放平台步骤见 [钉钉接入指南.md](./钉钉接入指南.md)。Web 配置摘要：

1. 在钉钉开放平台创建**企业内部应用**，添加 **Stream 模式机器人**，并**发布到企业**
2. 记录：**Client ID**、**Client Secret**、**Robot Code**（机器人编码）
3. Web → **设置** → **钉钉连接**，填写上述三项
4. **提醒用户 Staff ID**：你的钉钉员工 ID（多个用英文逗号分隔），用于到期提醒与 AI 分析报告推送  
   - 给机器人发一条测试消息后，可在任务数据里查看，或问管理员查通讯录
5. 点击 **测试连接** → **保存**（会自动重启钉钉桥）
6. 验证：`http://<NAS_IP>:17887/health` 中 `"bridge": true`

手机测试：与机器人单聊发送 `买牛奶`，Web 任务列表应出现新任务。

---

## 5. 配置大模型（可选）

不配置也能用：**统计页**仍可看数据，分析会使用**内置规则**生成报告。

启用 AI 分析：

1. Web → **设置** → **AI 大模型（周/月/年分析）**
2. 勾选 **启用**
3. 填写：

| 字段 | 示例 |
|------|------|
| API Base URL | `https://api.openai.com/v1` 或国内兼容地址 |
| API Key | `sk-...` |
| 模型 | `gpt-4o-mini` |

4. **测试连接** → **保存**
5. **统计** 页 → 选择周期 → **运行 AI 分析**

### 定时分析与钉钉推送

**设置** → **AI 分析定时与推送**：

- 默认：每周一 8:00 推上周报告、每月 1 日推上月、每年 1 月 1 日推上年
- 需已配置 **钉钉 reminderUserIds** 才会推送到钉钉

---

## 6. 配置到期提醒（可选）

**设置** → **提醒**：

- 启用钉钉提醒
- 设置延迟（截止后多久未完成任务时提醒）
- 保存

任务需有**截止日期**才会触发；接收人为任务发送者的 Staff ID，或 fallback 到 **reminderUserIds**。

---

## 7. 连接 Mac / Windows 瘦客户端

NAS 服务跑起来后：

```bash
cd notes-desk-client/mac && ./run.sh   # macOS
# 或 notes-desk-client/win 下 dotnet run
```

客户端 **设置** 填写：`http://<NAS_IP>:17887`（与 Docker 端口映射一致）。  
若 Docker 设置了 `NOTESDESK_TOKEN`，客户端也需填相同 Token。

详见 [notes-desk-client/说明.md](../../notes-desk-client/说明.md)。

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| `bridge: false` | 检查钉钉凭证、应用是否发布、容器日志 `docker logs notesdesk` |
| 推送报 `staffId.notExisted` | **提醒用户 Staff ID** 填错；需填 Staff ID 而非 userId |
| 登录页不出现 | 镜像过旧，需导入含 Web 登录的新版 tar |
| `SQLITE_CANTOPEN` | `sudo chown -R 999:999 /volume1/docker/notesdesk/data` |
| 钉钉收不到消息 | 同一 Client ID 只能一处 Stream；确认 Mac 无第二个 bridge |
