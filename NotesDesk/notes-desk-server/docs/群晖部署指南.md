# 群晖 Synology 部署指南

> NotesDesk Server · Phase 2  
> 目标：NAS 7×24 运行钉钉 Stream + SQLite + HTTP API（`:8080`）

**Mac 打镜像 → 导出 tar → 群晖导入**：见 [群晖Docker打包与部署.md](./群晖Docker打包与部署.md)

---

## 1. 前提

| 项 | 要求 |
|----|------|
| 群晖 | DSM 7+，已安装 **Container Manager** |
| 架构 | **x86（amd64）** 多数型号；ARM 型号需自行验证镜像 |
| 网络 | NAS 能访问外网（钉钉 Stream 出站连接） |
| 钉钉 | 开放平台应用：Client ID / Secret；提醒需 robotCode |

---

## 2. 目录规划

在 NAS 上创建：

```
/volume1/docker/notesdesk/
├── docker-compose.synology.yml   # 从本项目复制
├── Dockerfile                    # 从本项目复制
├── package.json                  # 构建需要（或整目录上传）
├── …（完整 notes-desk-server 源码）
└── data/                         # 持久化（compose 挂载）
    ├── config/
    │   ├── dingtalk.json
    │   └── llm.json              # 可选，AI 分析
    ├── store/
    │   └── notesdesk.db          # 自动创建
    └── backup/                   # 可选，手动或计划任务备份
```

**只需持久化 `data/`**，重建容器不丢任务。

## 3. 准备配置文件

### 3.1 dingtalk.json

复制 `config/dingtalk.config.example.json` → `data/config/dingtalk.json`：

```json
{
  "clientId": "你的 AppKey",
  "clientSecret": "你的 AppSecret",
  "robotCode": "机器人编码",
  "reminderUserIds": ["你的 staffId"]
}
```

### 3.2 llm.json（可选）

```json
{
  "enabled": true,
  "baseUrl": "https://api.openai.com/v1",
  "apiKey": "sk-...",
  "model": "gpt-4o-mini"
}
```

也可启动后通过 Web **设置** 页配置（推荐）。

## 4. Container Manager 部署

### 方式 A：项目（Compose）— 推荐

1. **文件管理器** 上传整个 `notes-desk-server` 到 `/volume1/docker/notesdesk/`
2. 确认 `data/config/dingtalk.json` 已就位
3. 打开 **Container Manager** → **项目** → **创建**
4. 项目名称：`notesdesk`
5. 路径：选择含 `docker-compose.synology.yml` 的目录
6. 来源：使用现有 `docker-compose.synology.yml`
7. **构建** 并 **启动**

### 方式 B：SSH 命令行

```bash
cd /volume1/docker/notesdesk/notes-desk-server
sudo docker compose -f docker-compose.synology.yml up -d --build
```

## 5. 验证

浏览器或局域网内电脑：

```
http://<NAS_IP>:8080/health
```

期望 JSON：

```json
{"ok":true,"bridge":true,"time":"..."}
```

- `bridge: true` → 钉钉 Stream 已连接（需有效 `dingtalk.json`）
- `bridge: false` → 检查配置与容器日志

```bash
docker logs notesdesk
```

测试收待办（模拟 ingest）：

```bash
curl -X POST http://<NAS_IP>:8080/ingest \
  -H 'Content-Type: application/json' \
  -d '{"text":"买牛奶","source":"manual"}'
```

---

## 6. 可选：API 访问令牌

在 compose 中设置环境变量：

```yaml
environment:
  NOTESDESK_TOKEN: your-secret-token
```

之后 API 请求需带：

```
Authorization: Bearer your-secret-token
```

`/health` 与 `/ingest` 不受令牌保护（内网使用）。

## 8. 从桌面版迁移数据

1. 复制 Mac 上 `~/Library/Application Support/NotesDeskIM/tasks.json`
2. 导入：

```bash
curl -X POST http://<NAS_IP>:8080/api/import/tasks \
  -H 'Content-Type: application/json' \
  -d @tasks.json
```

若启用了 `NOTESDESK_TOKEN`，加上 Authorization 头。

钉钉凭证：复制 `~/.notesdesk-im/dingtalk.json` → `data/config/dingtalk.json`

## 9. 常见问题

| 现象 | 处理 |
|------|------|
| `bridge: false` | 检查 `dingtalk.json` 路径、凭证、容器日志 |
| 端口冲突 | 修改 compose 中 `8080:8080` 左侧端口 |
| 重建容器后数据丢失 | 确认 volume 挂载 `/volume1/docker/notesdesk/data:/data` |
| Mac 上 `DeadlineExceeded` 拉 Docker Hub | 去掉 Dockerfile 首行 `# syntax=…`；或配置 Docker 镜像加速；**建议直接在 NAS 上 build** |
| NAS 上 build 也超时 | 群晖 Docker → 注册表 → 添加镜像加速器（如 `https://docker.m.daocloud.io`） |

## 10. 更新版本

```bash
cd /volume1/docker/notesdesk/notes-desk-server
git pull   # 或重新上传文件
sudo docker compose -f docker-compose.synology.yml up -d --build
```

`data/` 卷不变，任务保留。
