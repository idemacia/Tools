# 钉钉接入指南（NotesDesk Server / 群晖 Docker）

本文说明如何在钉钉开放平台创建 **Stream 模式机器人**，并在 NotesDesk **Web 设置** 或 `data/config/dingtalk.json` 中配置。

---

## 1. 重要：机器人类型

| 类型 | 在哪里 | 能否收消息 | NotesDesk |
|------|--------|------------|-----------|
| 群 **Webhook 自定义机器人** | 群设置 → 智能群助手 | ❌ 只能发不能收 | **不支持** |
| **企业内部应用机器人（Stream）** | 开放平台 → 应用 → 机器人 | ✅ | **必须用这个** |

---

## 2. 开放平台配置

### 2.1 创建企业内部应用

1. 打开 [钉钉开放平台](https://open.dingtalk.com/)
2. **应用开发** → **企业内部开发** → **创建应用**
3. 在 **凭证与基础信息** 记录：
   - **AppKey** → 对应 Web 里的 **Client ID**
   - **AppSecret** → 对应 **Client Secret**

### 2.2 添加 Stream 机器人

1. 进入应用 → **应用能力** → **添加应用能力** → **机器人**
2. 填写名称、头像
3. **消息接收模式** 选 **Stream 模式**（不要选 Webhook）
4. 保存后 → **版本管理与发布** → 创建版本 → **发布到企业**

未发布则手机找不到该机器人。

### 2.3 获取 Robot Code

在应用 **机器人** 配置页或应用详情中查看 **Robot Code**（机器人编码），填入 Web **钉钉连接**。

### 2.4 权限

**权限管理** 中开通机器人 / IM 相关权限（接收消息一般随机器人能力一并开通）。

---

## 3. 在 NAS / Web 中配置

### 方式 A：Web 设置（推荐）

1. 登录 `http://<NAS_IP>:17887`
2. **设置** → **钉钉连接**
3. 填写 Client ID、Client Secret、Robot Code、提醒用户 Staff ID
4. **测试连接** → **保存并重启桥**

### 方式 B：编辑配置文件

路径：`/volume1/docker/notesdesk/data/config/dingtalk.json`

```json
{
  "clientId": "你的 AppKey",
  "clientSecret": "你的 AppSecret",
  "robotCode": "机器人编码",
  "reminderUserIds": ["你的 staffId"]
}
```

复制模板：`config/dingtalk.config.example.json`。修改后重启容器：

```bash
sudo docker restart notesdesk
```

---

## 4. 获取 Staff ID（reminderUserIds）

**提醒**与 **AI 分析报告推送** 需要正确的 **Staff ID**（不是 userId）。

获取方式：

1. 给机器人发一条消息，在 Web 任务详情或导出 JSON 中查看 `dingtalkStaffId`（若消息回调带有）
2. 钉钉管理后台 → 通讯录 → 成员详情
3. 开放平台通讯录 API 查询

多个接收人用英文逗号分隔，例如：`0123456789,9876543210`

---

## 5. 手机端使用

**私聊（推荐测试）**

1. 钉钉 → **工作台** → 搜索你的应用 → 进入 **机器人单聊**
2. 发送：`买牛奶`、`完成 买牛奶`、`列表`

**群聊**

1. 群设置 → **机器人** → **从企业应用添加** → 选择你的应用机器人
2. 群里 **@机器人** 发文本

---

## 6. 验证

```bash
curl http://<NAS_IP>:17887/health
# 期望 "bridge": true

sudo docker logs notesdesk --tail 30
# 期望 [dingtalk-bridge] Stream connected
```

---

## 7. 常见问题

| 问题 | 原因 / 解决 |
|------|-------------|
| `bridge: false` | 凭证错误、应用未发布、NAS 无外网 |
| 503 后容器重启 | 旧版镜像问题；更新到新版（connect 失败会自动重试） |
| 消息进不来 | Mac 上是否还有第二个 Stream 连接同一 Client ID |
| `staffId.notExisted` | reminderUserIds 填错 ID 类型或员工不存在 |
| Secret 泄露 | 开放平台 **重置 AppSecret**，更新 Web 配置 |

---

## 8. 安全提示

- **不要**将 `dingtalk.json` 提交到公开 Git 仓库
- **不要**在 Issue、公众号、日志截图中暴露 Client Secret
- 泄露后立即在开放平台轮换 Secret
