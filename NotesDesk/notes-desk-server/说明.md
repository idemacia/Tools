# NotesDesk Server

群晖 Docker / 本地运行的 NotesDesk 后端：SQLite 持久化、钉钉 Stream、Web 管理、统计与 LLM 分析。

## 普通用户（群晖 NAS）— 只用 tar

**不需要** clone 源码、**不需要** `npm install` 或 `docker build`。

1. 从 [GitHub Releases](https://github.com/idemacia/Tools/releases) 下载 **`notesdesk-server.tar`**
2. 按 [release/README.md](release/README.md) 导入镜像并创建容器
3. 按 [docs/安装与配置指南.md](docs/安装与配置指南.md) 配置 Web、钉钉、大模型

钉钉开放平台步骤：[docs/钉钉接入指南.md](docs/钉钉接入指南.md)

## 开发者 — 源码与 build

### 本地开发

```bash
cd notes-desk-server
npm install --registry=https://registry.npmmirror.com
cp config/dingtalk.config.example.json data/config/dingtalk.json   # 可选
cp config/llm.config.example.json data/config/llm.json               # 可选
npm run build
npm run dev
```

改 TypeScript 时可另开终端：`npm run watch`（自动编译，`dev` 会热重启）。

默认 `http://127.0.0.1:8080`，数据目录 `./data`。

## Docker（本地 / 群晖）

```bash
cd notes-desk-server

# 本地（海外 / Docker Hub 正常）
docker compose up -d --build

# 国内 Mac：Docker Hub 超时时用国内镜像
docker compose -f docker-compose.yml -f docker-compose.cn.yml up -d --build

curl http://127.0.0.1:8080/health

# 群晖：见 docs/群晖部署指南.md
docker compose -f docker-compose.synology.yml -f docker-compose.cn.yml up -d --build
```

持久化目录：容器内 `/data`（compose 挂载到 `notesdesk-data` 或群晖 `/volume1/docker/notesdesk/data`）。

## 环境变量

| 变量 | 默认 | 说明 |
|------|------|------|
| `DATA_DIR` | `./data` | 配置与 SQLite |
| `PORT` | `8080` | HTTP 端口 |
| `TZ` | `Asia/Shanghai` | 时区 |
| `NOTESDESK_TOKEN` | 空 | 设置后 API 需 `Authorization: Bearer …` |

## API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/ingest` | 收消息（钉钉桥内部调用） |
| GET/POST/PATCH | `/api/tasks` | 任务 CRUD |
| GET | `/api/stats/weekly?year=&week=` | 周统计 |
| POST | `/api/stats/analyze` | LLM 分析报告 |
| GET/PUT | `/api/config/dingtalk` | 钉钉配置 |
| GET/PUT | `/api/config/llm` | 大模型配置 |
| POST | `/api/import/tasks` | 导入桌面版 JSON |
| GET | `/api/export/tasks?format=csv` | 导出 |
