# NotesDesk Widget 架构与开发文档

> 项目总览见 [NotesDesk README](../../README.md)。

## 1. 技术栈

| 层级 | 选型 |
|------|------|
| 语言 | Swift 5.9+ |
| Host UI | SwiftUI MenuBarExtra（`note.text`） |
| Widget UI | SwiftUI + WidgetKit `StaticConfiguration` |
| 读数据 | AppleScript → Notes.app（**仅 Host**） |
| Host 同步 | FSEvents（沙盒内常不可靠）+ **5s 轮询** |
| Host → Widget | localhost HTTP + Keychain + App Group 文件 |
| 构建 | XcodeGen + Xcode 16+（须签名） |
| 部署目标 | macOS 14+ |

## 3. 数据流

```
┌──────────────────── NotesDeskWidget Host（沙盒 App）────────────────────┐
│  MenuBarExtra (note.text)                                                │
│  SyncViewModel                                                           │
│    ├─ NotesWatcher（FSEvents，沙盒内对 Notes 目录常无效）                  │
│    ├─ 轮询 Task：每 hostSyncPollInterval（5s）调用 sync()                 │
│    ├─ NotesReader（AppleScript）                                         │
│    ├─ publishSnapshot → SharedStore / Keychain / WidgetSnapshotServer    │
│    └─ reloadWidgets()（内容指纹变化时）                                    │
└───────────────────────────────┬──────────────────────────────────────────┘
                                │
          ┌─────────────────────┼─────────────────────┐
          ▼                     ▼                     ▼
   localhost:19876      Keychain 共享组        App Group 文件
   /snapshot JSON        widget-current         widget-current.json 等
          │                     │                     │
          └─────────────────────┼─────────────────────┘
                                ▼
┌────────────── NotesDeskWidgetExtension（chronod 沙盒）──────────────────┐
│  NotesWidgetProvider.makeEntry() 读取顺序：                               │
│    1. WidgetSnapshotFetcher.fetchFromHost()                              │
│    2. KeychainSnapshotStore.load()                                       │
│    3. SharedStore.loadCurrentWidgetSnapshot() 等                         │
│  Timeline policy: .after(now + widgetTimelineRefreshInterval)  // 10s   │
└──────────────────────────────────────────────────────────────────────────┘
                                │ widgetURL
                                ▼
                    noteswidget://open?noteID=… → Host → Notes.app
```

## 4. 关键实现

### 4.1 SyncViewModel

- `@MainActor`，菜单栏绑定的同步中枢
- `start()`：启动 `NotesWatcher` + **5s 轮询** + 初始 `loadFolders()` / `sync()`
- `sync()`：AppleScript 读第一条 → `publishSnapshot()`
- `publishSnapshot()`：写入 SharedStore、Keychain、`WidgetSnapshotServer`；**内容指纹变化**时才 `reloadWidgets()`
- `reloadWidgets()`：`WidgetCenter.shared.reloadTimelines(ofKind: "NotesDeskWidget")`

### 4.2 WidgetSnapshotServer

- `NWListener` 绑定 loopback TCP 端口 **19876**
- 响应 `GET /snapshot`（任意请求均返回当前 JSON）
- 无 `@MainActor`；`NSLock` 保护 `snapshotData`

### 4.3 WidgetSnapshotFetcher / KeychainSnapshotStore

- Widget 扩展侧 `URLSession` 请求 `http://127.0.0.1:19876/snapshot`，超时 2s
- Keychain：`kSecAttrService = com.notesdesk.widget.snapshot`，access group `group.com.notesdesk.widget`
- **原因**：macOS 桌面 Widget 扩展实测**常无法读 App Group 文件**，localhost 与 Keychain 为主通道

### 4.4 SharedStore

- App Group ID：`group.com.notesdesk.widget`
- 双写：文件（`widget-current.json` 等）+ UserDefaults suite + CFPreferences
- Host 写入可靠；Extension 读取为兜底

### 4.5 NotesWidgetProvider

- `StaticConfiguration`（Widget 跟随 Host 所选文件夹，无 AppIntent 文件夹选择器）
- `widgetUseHardcodedTestData`：调试开关，跳过数据层验证 UI
- 时间线：**10s** 后再次 `getTimeline`（`AppConstants.widgetTimelineRefreshInterval`）

### 4.6 AppConstants

```swift
appGroupID = "group.com.notesdesk.widget"
urlScheme = "noteswidget"
widgetSnapshotPort = 19876
hostSyncPollInterval = 5          // Host AppleScript 轮询
widgetTimelineRefreshInterval = 10 // Widget 时间线兜底
widgetUseHardcodedTestData = false
```

## 5. Entitlements 摘要

**Host**（`Config/Host.entitlements`）：

- `com.apple.security.app-sandbox`
- `com.apple.security.application-groups` → `group.com.notesdesk.widget`
- `com.apple.security.automation.apple-events`
- `com.apple.security.temporary-exception.apple-events` → `com.apple.Notes`
- `com.apple.security.network.server`
- `keychain-access-groups`

**Widget Extension**（`Config/WidgetExtension.entitlements`）：

- `com.apple.security.app-sandbox`
- `com.apple.security.application-groups`
- `com.apple.security.network.client`
- `keychain-access-groups`

## 6. 构建与运行

```bash
cd notes-desk-widget
xcodegen generate
open NotesDeskWidget.xcodeproj
# Xcode：配置 Signing → ⌘R
```

**不可**用 `./run-host.sh` 联调 Widget（无 App Group 签名）。

调试：

```bash
./debug-check.sh
curl http://127.0.0.1:19876/snapshot
```

## 7. 限制与已知问题

| 问题 | 原因 | 当前缓解 |
|------|------|----------|
| Widget 读不到 App Group | chronod 沙盒 | localhost + Keychain |
| Host FSEvents 不触发 | App 沙盒 | 5s AppleScript 轮询 |
| 更新比浮窗慢 | reloadTimelines 限流 + 跨进程 | 10s Widget 时间线兜底 |
| 必须常驻 Host | 扩展无法读备忘录 | 菜单栏 Host；文档说明 |

## 8. 与浮窗版架构差异

浮窗版是 **1 个 App** 完成读数据 + 展示；小组件版因 WidgetKit 强制 Extension，必须拆成 **Host（= 苹果生态里的「备忘录 App」角色）+ Extension（= 小组件 UI）**。

详见 [架构文档](架构.md)。
