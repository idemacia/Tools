# NotesDesk Widget 需求说明

> 项目总览见 [NotesDesk README](../../README.md)。

## 1. 背景

在 [`notes-desk-panel`](../notes-desk-panel/) 浮窗 Demo 验证可行后，本项目尝试 **WidgetKit 桌面小组件** 形态，满足相同的备忘录只读展示需求，并使用户能以**系统原生小组件**的方式固定在桌面。

## 2. 与浮窗版差异

| 项 | 浮窗版 | 小组件版（本项目） |
|----|--------|---------------------|
| 展示 | NSPanel 可滚动 | Widget 固定尺寸、不可滚动 |
| 几百字 | 完整显示 | 大尺寸约 24 行，超出截断 |
| 进程 | **单 App** | **Host App + Widget Extension** |
| 菜单栏 note.text | 同一 App 的控制入口 | **Host 的数据同步进程**（必须常驻） |
| 同步 | FSEvents → 同进程 UI（约 1s） | Host 轮询 5s + Widget 时间线 10s |
| 点击 | 双击打开备忘录 | 点击 Widget → URL Scheme → Host → 备忘录 |
| 构建 | SPM `./run.sh` | Xcode 签名 + App Group |

## 3. 功能需求（MVP）

| ID | 需求 |
|----|------|
| F-01 | Widget 展示 Host 所选文件夹的第一条备忘录（**不在 Widget 内选文件夹**） |
| F-02 | 展示该文件夹 **默认排序第 1 条** 标题 + 正文 |
| F-03 | 大尺寸 Widget 尽量展示几百字预览 |
| F-04 | 备忘录修改后自动同步（Host **5s 轮询** + Widget **10s 时间线**兜底） |
| F-05 | 点击 Widget 在系统备忘录中打开该条 |
| F-06 | Host 菜单栏：选文件夹、立即同步并刷新 Widget、退出 |

## 4. 明确不做

- Widget 内编辑备忘录
- 锁定 / 加密备忘录
- **无 Host 运行的独立 Widget 同步**（Widget 沙盒无法读备忘录，见 [架构文档](架构.md)）
- 与苹果第一方小组件同等的近实时刷新（平台限制）

## 5. 验收标准

1. Xcode ⌘R 运行 Host，菜单栏选文件夹并同步，添加桌面 Widget 后显示第一条备忘录内容。
2. 修改备忘录后 **约 5～10 秒**内 Widget 更新（Host 保持运行）。
3. 点击 Widget 可打开对应笔记。
4. Host 退出后 Widget 显示最后一次缓存内容；Host 重新运行并同步后恢复更新。

## 6. 权限与隐私

- **自动化**（Host）：AppleScript 读取备忘录
- **App Group**：`group.com.notesdesk.widget`
- **localhost**：Host `network.server`（端口 19876），Extension `network.client`
- **Keychain 共享**：`keychain-access-groups`
- 数据不离开本机

## 7. 为何需要 Host？

Widget 扩展运行在系统 `chronod` 沙盒中，**不能** AppleScript 读备忘录、**不能**读 Notes 内部数据库。必须有一个有自动化权限的 Host 充当数据中转。详见 [架构文档](架构.md)。
