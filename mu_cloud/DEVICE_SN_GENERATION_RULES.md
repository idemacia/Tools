# 设备序列号(SN)生成规则文档

## 📋 概述

本文档定义了MuCloud智能设备的10位纯数字序列号（SN）生成规则。该SN将作为设备的唯一标识符，同时用作阿里云IoT平台的`device_name`字段。

**格式要求：** 10位纯数字，无字母、无特殊字符

---

## 🏗️ SN结构设计

### 完整结构（10位）

```
位置:  1-2    3-4      5-6      7-10
      ┌──┐  ┌──┐    ┌──┐    ┌────┐
      │类型│ │年月│  │批次│  │序号│
      └──┘  └──┘    └──┘    └────┘
示例: 10    2510     01     0001
结果: 1025100100001 (实际为10位)
```

### 结构说明

| 位置 | 字段名称 | 位数 | 说明 |
|------|---------|------|------|
| 1-2 | 设备类型码 | 2位 | 标识设备类型 |
| 3-4 | 生产年份 | 2位 | 年份后两位（如2025→25） |
| 5-6 | 生产月份 | 2位 | 月份（01-12） |
| 7-8 | 生产批次 | 2位 | 当月生产批次（01-99） |
| 9-10 | 序列号 | 2位 | 批次内序列号（01-99） |

## 📅 时间码规则

### 年份码（位置3-4）

取当前年份的**后两位数字**

| 年份 | 年份码 |
|------|-------|
| 2024年 | `24` |
| 2025年 | `25` |
| 2026年 | `26` |
| ... | ... |
| 2099年 | `99` |

### 月份码（位置5-6）

使用标准月份，**不足两位前补0**

| 月份 | 月份码 |
|------|-------|
| 1月 | `01` |
| 2月 | `02` |
| ... | ... |
| 10月 | `10` |
| 11月 | `11` |
| 12月 | `12` |

## 🔢 批次与序列号规则

### 生产批次（位置7-8）

- **取值范围：** 01-99
- **规则：** 每月第一批从01开始，依次递增
- **重置时机：** 每月1日重置为01
- **用途：** 区分同月不同批次的生产

**示例：**
- 2025年10月第1批：`01`
- 2025年10月第2批：`02`
- 2025年11月第1批：`01`（新月份重置）

### 序列号（位置9-10）

- **取值范围：** 01-99
- **规则：** 每批次从01开始，依次递增
- **重置时机：** 每个新批次重置为01
- **用途：** 标识批次内的具体设备

**容量计算：**
- 每批次最多：99台设备
- 每月最多：99批次 × 99台 = 9,801台设备
- 每年最多：9,801 × 12月 = 117,612台设备

## ✅ 生成流程（手动）

### Step 1: 确定设备类型

查表获取2位类型码：
- 智能LED灯 → `10`
- BloomIQ → `20`
- 智能插座 → `30`
- 智能窗帘 → `40`
- 红外控制器 → `50`

### Step 2: 获取当前日期

```
当前日期: 2025年10月7日
↓
年份码: 25
月份码: 10
```

### Step 3: 确定批次号

```
查询当月已有批次:
- 如果是本月第一批 → 01
- 如果已有批次 → 最大批次号+1
```

### Step 4: 分配序列号

```
查询当前批次已分配序列号:
- 如果是批次第一台 → 01
- 如果已有设备 → 最大序号+1
```

### Step 5: 组合生成

```
格式: [类型码][年码][月码][批次][序号]
示例: 10 + 25 + 10 + 01 + 01 = 1025100101
```

## 📊 批次管理建议

### 批次划分策略

#### 方案A：按时间划分（推荐）
```
上午批次: 01 (08:00-12:00)
下午批次: 02 (13:00-17:00)
夜班批次: 03 (18:00-22:00)
```

#### 方案B：按数量划分
```
每100台设备为一个批次
第1-100台: 批次01
第101-200台: 批次02
...
```

#### 方案C：按产线划分
```
产线A: 批次01, 03, 05, 07...（奇数）
产线B: 批次02, 04, 06, 08...（偶数）
```

## ⚠️ 注意事项

### 唯一性保证

✅ **确保唯一性的要素：**
- 设备类型不同 → 类型码不同
- 生产时间不同 → 年月码不同
- 同月不同批次 → 批次码不同
- 同批次不同设备 → 序列号不同

❌ **避免重复的关键：**
- 严格按顺序分配序列号
- 不要跳号分配
- 记录已使用的SN
- 建立SN分配日志

### 特殊情况处理

#### 1. 批次用尽（99批次）
```
解决方案：
- 等待下个月（批次自动重置）
- 或使用次月批次号提前生产
```

#### 2. 序列号用尽（99台）
```
解决方案：
- 创建新批次
- 序列号从01重新开始
```

#### 3. 年份达到2099年
```
解决方案：
- 升级SN规则（添加世纪位）
- 或使用新的编码体系
```

```sql
  end_sn VARCHAR(10),
```
