# 多设备支持架构说明

本文档说明为支持控制多个设备所需完成的架构改造工作。

## 一、现状分析

### 1.1 当前架构

```
┌──────────────────────────────────────────────────────────────────┐
│ index (首页)                                                       │
│ devices: [ { id: 'smartled', path: '/pages/smartled/smartled' },   │
│            { id: 'socket',   path: '/pages/socket/socket' } ]      │
│ → 按设备类型导航，不传递 deviceId                                    │
└──────────────────────────────────────────────────────────────────┘
                                    │
         ┌──────────────────────────┴──────────────────────────┐
         ▼                                                      ▼
┌─────────────────────┐                              ┌─────────────────────┐
│ smartled 页面        │                              │ socket 页面          │
│ device_sn ← config  │                              │ device_sn ← config  │
│ deviceNameSmartled  │                              │ deviceNameSocket    │
│ (单设备)             │                              │ (单设备)             │
└─────────┬───────────┘                              └─────────┬───────────┘
          │                                                     │
          └─────────────────────┬───────────────────────────────┘
                                ▼
                    ┌───────────────────────┐
                    │ device-service        │
                    │ 所有方法接收 deviceId  │  ← API 层已支持多设备
                    │ getDeviceStatus(id)   │
                    │ controlDevice(id,cmd) │
                    └───────────────────────┘
```

### 1.2 已具备多设备能力的部分

| 模块 | 现状 | 说明 |
|------|------|------|
| device-service | ✅ 已支持 | 所有方法接收 deviceId 参数，按设备维度调用 |
| aliyun-connector | ✅ 已支持 | queryDeviceData、invokeService、queryDeviceStatus 均接收 deviceId |
| 阿里云 OpenAPI | ✅ 已支持 | 按 ProductKey + DeviceName 操作，天然多设备 |

### 1.3 需改造的部分

| 模块 | 现状 | 改造点 |
|------|------|--------|
| iot-config.js | 每种类型一个 deviceName | 改为设备列表或支持数组 |
| index 首页 | 按设备类型导航 | 改为按设备实例列表展示，或增加设备选择入口 |
| smartled / socket 页面 | 单设备，deviceId 来自 config | 支持从路由参数/选择器获取 deviceId |
| 设备列表来源 | 静态配置 | 可选：动态从阿里云 QueryDeviceList 拉取 |

## 二、架构改造方案

### 方案 A：配置驱动（静态多设备）

**适用**：设备数量固定、用户自行配置设备列表

#### 2.1 配置层

**文件**：`config/iot-config.js`

```javascript
// 改造前
deviceNameSmartled: 'led_01',
deviceNameSocket: 'socket_01',

// 改造后
deviceListSmartled: ['led_01', 'led_02', 'led_03'],
deviceListSocket: ['socket_01', 'socket_02'],
// 或统一设备列表（含类型）
deviceList: [
  { type: 'smartled', deviceName: 'led_01', displayName: '客厅灯' },
  { type: 'smartled', deviceName: 'led_02', displayName: '卧室灯' },
  { type: 'socket',   deviceName: 'socket_01', displayName: '玄关插座' },
],
```

#### 2.2 首页

**文件**：`pages/index/index.js`、`pages/index/index.wxml`

- 从 config 读取 `deviceList`（或按类型分别读取）
- `data.devices` 改为设备实例列表，每项包含：`id`、`name`、`deviceName`、`type`、`path`
- 导航时通过 URL 参数传递 `deviceName`，例如：`/pages/smartled/smartled?deviceName=led_01`

#### 2.3 设备控制页面

**文件**：`pages/smartled/smartled.js`、`pages/socket/socket.js`

- `onLoad(options)` 中读取 `options.deviceName`
- 若未传，回退到 `iotConfig.deviceNameSmartled` 等（保持单设备兼容）
- 所有 `getDeviceId()` 改为返回当前选中设备的 deviceName

#### 2.4 设备选择器（同类型多设备时）

- 在 smartled / socket 页面顶部增加设备切换 UI（tabs、下拉、列表）
- `data.deviceList` 存当前类型下的设备列表
- `data.currentDevice` 为当前选中设备
- 切换时更新 `currentDevice`，并重新 `fetchDeviceStatus`、`startStatusPolling`

---

### 方案 B：动态拉取（云端设备列表）

**适用**：设备数量不固定、需从阿里云实时获取

#### 2.1 新增设备列表服务

**文件**：`services/device-service.js` 或新建 `services/device-list-service.js`

- 调用阿里云 `QueryDevice` / `QueryDeviceList` 获取产品下设备列表
- 返回结构：`[{ DeviceName, ProductKey, NickName?, Status? }]`
- 可按 ProductKey 过滤，支持多产品

#### 2.2 新增 aliyun-connector 能力

**文件**：`utils/aliyun-connector.js`

```javascript
// 新增方法
async queryDeviceList(productKey, pageSize, currentPage) {
  const params = {
    Action: 'QueryDevice',
    ProductKey: productKey,
    PageSize: pageSize,
    CurrentPage: currentPage
  };
  // 调用 aliIot-sdk.request
}
```

#### 2.3 首页与设备页

- 首页 onLoad 时调用 `deviceService.queryDeviceList()` 获取设备列表
- 按设备类型分组或扁平展示，点击进入控制页并传递 `deviceName`
- 设备控制页逻辑同方案 A

---

### 方案 C：混合模式（配置 + 动态）

- 优先使用 config 中的 `deviceList`
- 可选：提供「从云端刷新」按钮，调用 `QueryDeviceList` 更新列表并写入本地缓存（wx.setStorageSync）

## 三、详细改造清单

### 3.1 配置层 (config/iot-config.js)

| 改造项 | 说明 |
|--------|------|
| 新增 deviceListSmartled | 智能灯设备名数组，如 `['led_01','led_02']` |
| 新增 deviceListSocket | 插座设备名数组 |
| 或统一 deviceList | `[{ type, deviceName, displayName }]` |
| 保留 deviceName / deviceNameSmartled 等 | 向后兼容单设备配置 |

### 3.2 首页 (pages/index/)

| 改造项 | 说明 |
|--------|------|
| data.devices | 改为从 config 或服务层获取设备实例列表 |
| 每项数据结构 | `{ id, name, deviceName, type, path }` |
| navigateToDevice | 跳转时携带 `?deviceName=xxx` |
| 可选 | 展示设备在线状态（需调用 BatchGetDeviceState） |

### 3.3 设备控制页 (pages/smartled/, pages/socket/)

| 改造项 | 说明 |
|--------|------|
| onLoad(options) | 读取 `options.deviceName` 作为当前设备 |
| 无 deviceName 时 | 使用 config 单设备配置，保持兼容 |
| data.deviceList | 同类型设备列表（用于切换） |
| data.currentDevice | 当前选中设备名 |
| 设备切换 UI | tabs / picker / 下拉，切换时更新 currentDevice 并刷新状态 |
| fetchDeviceStatus | 使用 `this.data.currentDevice` 或 `getDeviceId()` |
| startStatusPolling | 轮询时使用当前设备 ID |
| 页面标题 | 可显示设备别名（如「客厅灯」） |

### 3.4 服务层 (services/device-service.js)

| 改造项 | 说明 |
|--------|------|
| 现有 API | 无需改动，已支持 deviceId |
| 可选新增 | `queryDeviceList(productKey, pageSize, page)` |
| 可选新增 | `batchGetDeviceState(deviceNames)` |

### 3.5 连接器层 (utils/aliyun-connector.js)

| 改造项 | 说明 |
|--------|------|
| 现有方法 | 无需改动 |
| 可选新增 | `queryDeviceList` 封装 QueryDevice API |
| 构造函数 | 当前要求 deviceName，可改为可选（仅作默认值） |

### 3.6 阿里云 API 依赖

| API | 用途 | 是否必选 |
|-----|------|----------|
| QueryDevicePropertyStatus | 单设备属性查询 | 已有 |
| SetDeviceProperty | 单设备写属性（智能灯默认） | 已有 |
| InvokeThingService | 单设备调服务（插座等） | 已有 |
| QueryDevice | 产品下设备列表 | 方案 B 需新增 |
| BatchGetDeviceState | 批量设备在线状态 | 可选，用于首页状态展示 |

## 四、路由与参数传递

### 4.1 首页到控制页

```
index.wxml:
  wx:for="{{devices}}"
  bindtap="navigateToDevice"
  data-path="/pages/smartled/smartled"
  data-device-name="{{item.deviceName}}"

index.js:
  navigateToDevice(e) {
    const { path, deviceName } = e.currentTarget.dataset;
    wx.navigateTo({
      url: deviceName ? `${path}?deviceName=${deviceName}` : path
    });
  }
```

### 4.2 控制页接收

```javascript
onLoad(options) {
  const deviceName = options.deviceName
    || iotConfig.deviceNameSmartled
    || iotConfig.deviceName
    || wx.getStorageSync('deviceNameSmartled');
  this.setData({ 'device.device_sn': deviceName, currentDevice: deviceName });
  // 若同类型多设备，可加载 deviceList 供切换
  const list = iotConfig.deviceListSmartled || (deviceName ? [deviceName] : []);
  this.setData({ deviceList: list });
  // ...
}
```

## 五、同类型多设备切换 UI 示例

### 5.1 顶部 tabs

```html
<view class="device-tabs">
  <view wx:for="{{deviceList}}" wx:key="*this"
        class="tab {{currentDevice === item ? 'active' : ''}}"
        bindtap="switchDevice" data-device="{{item}}">
    {{item.displayName || item}}
  </view>
</view>
```

### 5.2 页面逻辑

```javascript
switchDevice(e) {
  const device = e.currentTarget.dataset.device;
  const deviceName = typeof device === 'string' ? device : device.deviceName;
  this.setData({ currentDevice: deviceName, 'device.device_sn': deviceName });
  this.fetchDeviceStatus();
  this.startStatusPolling(); // 若已有定时器，需先 clear 再 start
}
```

## 六、实施建议

1. **阶段一**：实现方案 A（配置驱动），先支持多设备配置与路由传参，控制页按 deviceName 工作。
2. **阶段二**：在控制页增加设备切换 UI，支持同类型多设备切换。
3. **阶段三**（可选）：实现方案 B 或 C，增加 `QueryDeviceList`，支持动态设备列表与在线状态展示。
