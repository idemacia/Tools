## 二、目录职责

| 目录/文件 | 职责 |
|-----------|------|
| `pages/index/` | 首页，设备类型入口 |
| `pages/smartled/` | 智能灯控制页面 |
| `pages/socket/` | 智能插座控制页面 |
| `components/header/` | 通用头部组件（标题、返回） |
| `services/device-service.js` | 设备服务：封装阿里云 API，提供业务级方法 |
| `utils/aliyun-connector.js` | 阿里云连接器：底层 HTTP 调用 |
| `utils/aliIot-sdk.js` | 阿里云 IoT SDK（签名、请求） |
| `config/iot-config.js` | 阿里云配置（AccessKey、ProductKey、DeviceName） |

## 三、数据流

### 3.1 获取设备状态

```
页面 onLoad
  → deviceService.getDeviceStatus(deviceId)
  → connector.queryDeviceData(deviceId)
  → aliIot-sdk.request(QueryDevicePropertyStatus)
  → 阿里云 API
  → 解析 PropertyStatusInfo
  → 页面 setData 更新 UI
```

### 3.2 控制设备

**智能灯（默认）**：页面仍传 `{ ledswitch, lightBrightness, hue, ... }`，`device-service` 映射为 TSL 字段后调用 **`SetDeviceProperty`**（`Items` 如 `{"power":1,"brightness":50}`）。若 `iot-config.smartledUseInvokeThingService === true`，则改为 **`InvokeThingService`**，Identifier 见 `smartledThingServiceIdentifier`（如 `set`）。

**智能插座**：`controlDevice` 识别 `plugSwitch` 等字段后走 **`InvokeThingService`**，Identifier 见 `socketThingServiceIdentifier`。

```
用户操作
  → deviceService.controlDevice(deviceId, command)
  → connector.setDeviceProperty(deviceId, items)   // 智能灯默认
  → 或 connector.invokeService(deviceId, args, identifier)  // 插座或配置为服务调用时
  → aliIot-sdk.request(...)
  → 阿里云 API
  → 页面 showToast
```

## 四、配置说明

`config/iot-config.js` 提供：

- `productKey`、`accessKeyId`、`accessKeySecret`、`endpoint`
- `deviceName`、`deviceNameSmartled`、`deviceNameSocket`
- `smartledUseInvokeThingService`：智能灯是否用 `InvokeThingService`（默认 `false`，用 `SetDeviceProperty`）
- `smartledThingServiceIdentifier`、`socketThingServiceIdentifier`：服务 Identifier
- `iotInstanceId`：企业版实例 ID（公共实例可留空）

配置优先级：`wx.getStorageSync('iotConfig')` 合并覆盖 > 文件内默认值。
